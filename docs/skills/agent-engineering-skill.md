# Agent Engineering Skill

## 触发场景

当任务涉及 AI Agent、RAG、Tool Calling、多 Agent、提示词工程、模型接入、上下文记忆、知识库问答时使用本 skill。

如果任务涉及 OA AI 操作面板、AI plan/execute mock、AI 动作权限、AI 审计时间线，请同时读取 `docs/skills/oa-workbench-skill.md`。

## Agent 设计模板

新增 Agent 前先写清楚：

```text
Agent 名称：
业务目标：
用户输入：
模型输出：
可用工具：
禁止动作：
鉴权范围：
失败兜底：
观测指标：
测试用例：
```

## 最小 Agent 合约

每个 Agent 至少包含：

- `id`：稳定标识。
- `name`：用户可见名称。
- `description`：能力说明。
- `systemPrompt`：系统提示词。
- `tools`：可用工具列表。
- `memoryPolicy`：记忆策略。
- `safetyPolicy`：安全策略。
- `outputSchema`：结构化输出约束。

## OA AI 约束

当前 OA AI 是 mock 操作面板，不是真实自主执行系统：

- 用户能做什么，AI 最多只能做什么。
- 用户不能做的事，AI 不能代做。
- 普通员工不得执行审批、删除、权限修改、敏感导出。
- 高风险动作必须二次确认。
- 执行必须有审计记录。
- 失败时必须可解释，并允许 fallback mock。
- 当前阶段不调用真实 LLM，不操作真实数据库。

## RAG Agent 流程

1. 判断是否需要检索。
2. 生成检索 query。
3. 按 userId/tenantId 权限过滤。
4. 向量检索 + 关键词召回。
5. 重排或打分裁剪。
6. 组装上下文，保留来源。
7. 生成回答。
8. 输出引用和不确定性说明。

## Tool Agent 流程

1. 识别用户意图。
2. 判断是否需要工具。
3. 选择白名单工具。
4. 生成结构化参数。
5. 服务端校验参数和权限。
6. 执行工具。
7. 把工具结果压缩成模型可用上下文。
8. 输出用户可理解结果。

## Prompt 要求

系统提示词必须简洁、明确、可测试：

- 说明角色和边界。
- 说明何时调用工具。
- 说明输出格式。
- 说明无法确定时如何回答。
- 说明安全拒绝条件。

禁止：

- 硬编码密钥。
- 绕过鉴权。
- 绕过数据隔离。
- 要求模型“无论如何都执行”。

## 评估用例

每个 Agent 至少准备：

- 3 条正常问题。
- 2 条知识库缺失问题。
- 2 条越权或敏感问题。
- 1 条工具参数错误问题。
- 1 条模型应拒答问题。
- 1 条长上下文问题。
