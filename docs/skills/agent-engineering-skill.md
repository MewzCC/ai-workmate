# Agent Engineering Skill

## 触发场景

当任务涉及 AI Agent、RAG、Tool Calling、多 Agent、提示词工程、模型接入、上下文记忆、知识库问答时使用本 skill。

## Agent 设计模板

新增 Agent 前先写清楚：

```text
Agent 名称：
业务目标：
用户输入：
模型输出：
可用工具：
禁止动作：
鉴权范围：
失败兜底：
观测指标：
测试用例：
```

## 最小 Agent 合约

每个 Agent 至少包含：

- `id`：稳定标识。
- `name`：用户可见名称。
- `description`：能力说明。
- `systemPrompt`：系统提示词。
- `tools`：可用工具列表。
- `memoryPolicy`：记忆策略。
- `safetyPolicy`：安全策略。
- `outputSchema`：结构化输出约束。

## RAG Agent 流程

1. 判断是否需要检索。
2. 生成检索 query。
3. 按 userId/tenantId 权限过滤。
4. 向量检索 + 关键词召回。
5. 重排或打分裁剪。
6. 组装上下文，保留来源。
7. 生成回答。
8. 输出引用和不确定性说明。

## Tool Agent 流程

1. 识别用户意图。
2. 判断是否需要工具。
3. 选择白名单工具。
4. 生成结构化参数。
5. 服务端校验参数和权限。
6. 执行工具。
7. 把工具结果压缩成模型可用上下文。
8. 输出用户可理解结果。

## Prompt 要求

系统提示词必须简洁、明确、可测试：

- 说明角色和边界。
- 说明何时调用工具。
- 说明输出格式。
- 说明无法确定时如何回答。
- 说明安全拒绝条件。

示例骨架：

```text
你是 AI WorkMate 的企业知识助手。
你只能基于用户有权限访问的资料和工具结果回答。
当问题需要企业资料时，先检索知识库。
当资料不足时，明确说明缺少信息，不要编造。
输出使用简洁中文，必要时列出引用来源。
```

## 评估用例

每个 Agent 至少准备：

- 3 条正常问题。
- 2 条知识库缺失问题。
- 2 条越权或敏感问题。
- 1 条工具参数错误问题。
- 1 条模型应拒答问题。
- 1 条长上下文问题。

## 上线检查

- 有清晰的工具白名单。
- 有最大上下文长度和最大工具调用次数。
- 有超时与重试策略。
- 有审计日志。
- 有成本与延迟指标。
- 有回归测试样例。
